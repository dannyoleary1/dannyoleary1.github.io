Projects

Semi-Autonomous Trend Detection using Real Time Twitter Data

This project is a web application which a user can use to detect trends on the topics they are interested in. The web app works in a semi-autonomous way and will do detection of trends in the background. It will detect trends in real time. The trend can be monitored over time and isn’t removed until the user wants it removed. Some of the features of the web application include:

  • Semi-Autonomous trend detection on real time Twitter data.
  • Adding the topics to detect trends for based on the user’s preference.
  • Adding followers/Twitter users to detect topics that you may wish to include. A user of the web application can add Twitter users that they are interested in to help with picking the right topics for them. This feature will report back a list of potential topics which can then be added/declined.
  • Collect old tweets based on a topic to detect what some of the keywords on a topic were. An example of this would be to collect tweets on ‘youtube’ after it has had an outage. This feature will return the most popular words based on this, which may be: ‘down’, ‘outage’, or ‘hacked’. This gives a user a suggestion on what topics may be useful to get a broad range of trends e.g. outage would be a good topic to include if you were interested in telling when websites go down.
  • Table views. This is a table view of the data that has been collected over time with the potential trends displayed in red. The table can be changed based on the topic you want.
  • Display information on charts. The charts that are currently supported are: spline, line, bar, area, and area spline.

Stocks and Shares - Portfolio

This web application is designed for a user who is interested in keeping track of the stocks and shares that they own. This was done as part of a college assignment, and is made for one individual in particular with information provided in the specification. Note that it is no longer connected to the API to collect live data so that is no longer running. Some of the features that this web application contains are:

  • Add Stock: This allows the user to add a stock to the web application. The information that will be needed is the price at which it was bought, and how much it cost.
  • Reset Data: This sets the data back to the default state that is located in the MongoDB.
  • Select Test Case: The test cases are used as part of the assignment to ensure that the calculations made are in coordination with the specification.
  • Sell Stock: Allows the user to sell at the current price of the stock.
  • What if: This gives scenarios for the user if they were to do a certain action. For example if the user wanted to liquidate everything they own, the 'what if' section would give the profit/loss of doing so. Another feature that is contained within 'what if' is to sell only the stocks that are currently in profit.
  • Search: A dynamic search that will filter as the user types.
  • Transaction history: This keeps track of all the transactions that the user has had, along with the date it was bought and the date sold.

Keystroke dynamics

Keystoke dynamics or typing dynamics is the method of identifying or attempting to confirm the identity of a person. This research project identifies the usefulness of this biometric and how well it works. This basic idea is to use Machine Learning to identify patterns in the way a user types, for example how long they hold a key down. In the PDF, it explains the project in much more detail and the web app is only designed for demonstration purposes.
